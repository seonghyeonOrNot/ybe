## 1. 정책 정의

### 1.1 연차 상태값 정의

연차휴가의 생명주기를 관리하기 위한 상태값을 다음과 같이 정의한다.

| 상태코드 | 상태명 | 설명 | 비고 |
|----------|--------|------|------|
| `GENERATED` | 발생 | 연차가 새로 발생한 상태 | 1년 미만: 월별 발생, 1년 이상: 일괄 발생 |
| `PLANNED` | 사용예정 | 계획서에 등록되었으나 실제 사용 전 | **신규** - 계획서 제출과 차감 분리 |
| `USED` | 사용 | 실제 연차를 사용 완료한 상태 | 사용일 도래 시 자동 전환 |
| `CANCELLED` | 취소 | 사용예정이었으나 취소된 상태 | PLANNED에서만 전환 가능 |
| `EXPIRED` | 소멸 | 사용 기한이 만료되어 소멸된 상태 | 발생일로부터 1년 경과 |
| `CARRIED_OVER` | 이월 | 다음 연도로 이월된 상태 | 촉진 대상에서 제외 |
| `ADVANCED` | 당겨쓰기 | 미래 발생 예정 연차를 선사용 | 차감 대상에 포함 |

#### 연차 상태 전이 조건

```
[GENERATED] ──┬──(계획서 제출)──→ [PLANNED] ──┬──(사용일 도래)──→ [USED]
              │                               │
              │                               └──(일정 취소)──→ [CANCELLED] ──→ [GENERATED]
              │
              ├──(사용기한 만료)──→ [EXPIRED]
              │
              ├──(이월 처리)──→ [CARRIED_OVER]
              │
              └──(선사용 신청)──→ [ADVANCED] ──→ [USED]
```

### 1.2 촉진 상태값 정의

연차촉진 프로세스의 진행 단계를 관리하기 위한 상태값을 다음과 같이 정의한다.

| 상태코드 | 상태명 | 설명 | 전이 조건 |
|----------|--------|------|-----------|
| `NOT_STARTED` | 미시작 | 촉진 대상이지만 아직 시작되지 않음 | 촉진 대상자 선정 시 |
| `FIRST_NOTIFIED` | 1차발송 | 1차 촉진 고지 발송 완료 | 소멸 6개월 전 자동 또는 수동 발송 |
| `FIRST_SUBMITTED` | 1차제출 | 사용자가 1차 계획서 제출 완료 | 10일 내 계획서 제출 |
| `FIRST_OVERDUE` | 1차미제출 | 1차 계획서 미제출로 기한 초과 | 발송 후 10일 경과 |
| `SECOND_NOTIFIED` | 2차발송 | 2차 촉진 지정통보 발송 완료 | 소멸 2개월 전 + 미사용연차 존재 시 |
| `SECOND_SUBMITTED` | 2차제출 | 사용자가 2차 지정일정 확인 완료 | 사용자 확인 완료 |
| `SECOND_OVERDUE` | 2차미확인 | 2차 지정통보 미확인으로 기한 초과 | 발송 후 미확인 |
| `COMPLETED` | 완료 | 촉진 절차 정상 완료 | 모든 연차 사용 또는 소멸 |
| `EXEMPTED` | 면제 | 촉진 면제 대상 | 퇴사, 휴직 등 |

#### 촉진 상태 전이 조건

```
[NOT_STARTED] ──(소멸 6개월 전)──→ [FIRST_NOTIFIED] ──┬──(10일 내 제출)──→ [FIRST_SUBMITTED]
                                                      │
                                                      └──(10일 경과)──→ [FIRST_OVERDUE]

[FIRST_SUBMITTED/FIRST_OVERDUE] ──(소멸 2개월 전 + 잔여연차)──→ [SECOND_NOTIFIED]
                                                                │
                                                                ├──(확인 완료)──→ [SECOND_SUBMITTED]
                                                                │
                                                                └──(미확인)──→ [SECOND_OVERDUE]

[SECOND_SUBMITTED/SECOND_OVERDUE] ──(소멸일 도래)──→ [COMPLETED]

* 모든 상태에서 [EXEMPTED]로 전환 가능 (퇴사, 휴직 등)
```

### 1.3 촉진 우선순위 정책

| 우선순위 | 구분 | 설명 |
|----------|------|------|
| 1 | 당해년도 발생 연차 | 촉진 대상 (이월연차 제외) |
| 2 | 당겨쓴 연차 | 미래 발생분에서 차감 반영 |
| 3 | 승인된 미사용연차 | 1차 촉진 시점 기준 포함 (PLANNED 상태) |

#### 촉진 대상 연차 산정 공식

```
촉진 대상 연차 = 당해년도 발생 연차
               - 사용 완료 연차(USED)
               - 당겨쓴 연차(ADVANCED)
               + 사용예정 연차(PLANNED)  ※ 1차 촉진 시점 기준

※ 제외 항목: 이월연차(CARRIED_OVER)
```

## 4. 마이그레이션 영향 분석

### 4.1 기존 데이터 처리 규칙

#### 계획서 = 연차차감 분리 마이그레이션

| 구분 | 기존 상태 | 신규 상태 | 처리 방법 |
|------|-----------|-----------|-----------|
| 사용 완료 | 사용(USED) + 차감됨 | USED | 변경 없음 |
| 계획서 제출 + 미사용 | 사용예정 + 차감됨 | PLANNED + 미차감 | **원복 필요** |
| 계획서 제출 + 사용일 도래 | 사용예정 + 차감됨 | USED + 차감됨 | USED로 상태 변경 |

```
마이그레이션 로직:
1. 상태가 '사용예정'이고 사용일이 미도래인 연차 추출
2. 차감된 연차수를 잔여연차에 원복
3. 상태를 PLANNED로 변경 (차감 해제)
4. 로그 기록 및 감사 추적
```

#### 이월연차 촉진대상 제외 마이그레이션

| 구분 | 기존 처리 | 신규 처리 | 처리 방법 |
|------|-----------|-----------|-----------|
| 이월연차 촉진 대상 | 촉진 대상에 포함 | 촉진 대상에서 제외 | **제외 처리** |
| 이월연차 이미 촉진 발송 | 1차/2차 발송 완료 | 촉진 상태 EXEMPTED | **면제 처리** |
| 당해년도 연차 | 촉진 대상 | 촉진 대상 유지 | 변경 없음 |

```
마이그레이션 로직:
1. 연차 유형이 '이월'인 항목 추출
2. 해당 연차의 촉진 상태를 EXEMPTED로 변경
3. 촉진 대상 연차수 재계산
4. 관리자에게 변경 내역 리포트 제공
```

### 4.2 입사일 변경 시 데이터 처리

| 구분 | 기존 동작 | 신규 동작 |
|------|-----------|-----------|
| 촉진 데이터 | 기존 데이터 유지 (오류 발생) | **알림 + 초기화 옵션 제공** |
| 연차 발생일 | 수동 수정 필요 | **자동 재계산** |
| 1년 미만/이상 전환 | 수동 처리 | **자동 판단 및 적용** |

#### 입사일 변경 처리 플로우

```
1. 관리자가 입사일 변경 요청
2. 시스템이 영향 범위 분석
   - 연차 발생 일정 변경 여부
   - 촉진 일정 변경 여부
   - 소멸 일정 변경 여부
3. 관리자에게 영향 내역 알림
4. 관리자 선택:
   a) 촉진 데이터 초기화 → 새 입사일 기준 재계산
   b) 기존 데이터 유지 → 수동 관리
5. 선택에 따른 데이터 처리
6. 변경 이력 로그 기록
```

### 4.3 마이그레이션 체크리스트

| 단계 | 항목 | 담당 | 완료 기준 |
|------|------|------|-----------|

| 2 | 계획서 차감 분리 마이그레이션 | 개발팀 | PLANNED 상태 원복 완료 |

| 4 | 데이터 정합성 검증 | QA팀 | 검증 리포트 승인 |
| 5 | 관리자 변경 내역 공지 | PM | 공지 발송 완료 |
| 6 | 사용자 가이드 업데이트 | PM | 가이드 배포 완료 |

#### 제61조 연차휴가 사용촉진 요건

| 촉진 단계 | 시점 | 사용자(회사) 의무 | 근로자 의무 |
|-----------|------|-------------------|-------------|
| 1차 촉진 | 소멸 6개월 전 | 미사용 연차 일수 알림 + 사용시기 지정 요청 | 10일 이내 사용 계획서 제출 |
| 2차 촉진 | 소멸 2개월 전 | 미사용 연차 사용 시기 지정 통보 | 지정된 시기에 사용 |

### 5.2 용어 정의

| 용어 | 정의 |
|------|------|
| 연차 | 연차유급휴가의 약칭 |
| 월차 | 1년 미만 근로자에게 매월 발생하는 연차 (법률상 연차와 동일) |
| 촉진 | 연차휴가 사용촉진 제도 |
| 이월연차 | 전년도 미사용 연차를 당해년도로 이월한 연차 |
| 당겨쓰기 | 미래 발생 예정 연차를 선사용하는 것 |
| 소멸 | 사용 기한(발생일로부터 1년) 경과로 연차가 무효화되는 것 |
